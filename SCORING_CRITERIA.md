# 评分标准 (Scoring Criteria)

本文档定义了不同比赛类型的评分维度和权重分配。

## 评分范围

**重要变更**：每个评分维度的分数范围等于其权重百分比。

例如：
- 权重 30% 的维度，分数范围为 **0-30 分**
- 权重 25% 的维度，分数范围为 **0-25 分**
- 权重 15% 的维度，分数范围为 **0-15 分**

这样设计的优点：
1. **直观明了**：权重即最大分数，评委更容易理解
2. **简化计算**：各维度分数相加即为总分（满分100分）
3. **防止错误**：输入限制与权重一致，减少评分错误

## 评分维度详细说明

### 1. 动作难度 (Action Difficulty)
- 技术动作的难度系数
- 动作的复杂程度
- 高难度动作的完成质量
- 风险性动作的执行

### 2. 舞台艺术 (Stage Artistry)
- 舞台表现力
- 情感表达
- 艺术感染力
- 舞台魅力

### 3. 动作创意 (Action Creativity)
- 编排的创新性
- 动作的独特性
- 创意元素的运用
- 整体设计的新颖度

### 4. 动作流畅 (Action Fluency)
- 动作之间的衔接
- 整体表演的连贯性
- 节奏把控
- 流畅度

### 5. 服装造型 (Costume Styling)
- 服装设计
- 整体造型效果
- 与主题的契合度
- 视觉呈现

### 6. 动作互动 (Action Interaction)
- 团队成员之间的配合
- 互动的默契度
- 协同动作的完成度
- 团队整体性

## 评分规则

### 基本规则
1. **分数范围**：每个维度的分数范围为 0 到其权重百分比
   - 例如：30% 权重的维度，分数范围为 0-30 分
2. 评委需要为所有维度打分
3. 分数可以包含小数点（精确到小数点后两位）
4. 每位评委对每位选手只能提交一次评分
5. **总分计算**：各维度分数直接相加，满分为 100 分

### 评分流程
1. 评委登录系统
2. 选择比赛
3. 选择选手
4. 根据比赛类型，系统自动显示对应的评分维度
5. 为每个维度打分（范围为 0 到该维度的权重百分比）
6. 系统自动验证分数是否在有效范围内
7. 提交评分

### 分数计算
- **直接相加**：各维度分数直接相加得到总分
- **满分 100**：所有维度满分相加等于 100 分
- **无需加权**：由于分数范围已等于权重，无需额外计算
- 多位评委的分数会进行统计分析（平均分、最高分、最低分等）

### 评分示例

**个人舞台赛示例**：
- 动作难度：28/30
- 舞台艺术：23/25
- 动作创意：18/20
- 动作流畅：13/15
- 服装造型：8/10
- **总分**：90/100

## 技术实现

### 前端显示
- 评分表单根据比赛类型动态显示对应的评分维度
- 每个维度标签旁显示权重百分比
- 使用蓝色徽章显示权重（如：30%）
- 输入框的 placeholder 显示该维度的分数范围（如：0-30）
- 输入验证确保分数不超过该维度的最大值

### 数据存储
```sql
-- scores 表结构
CREATE TABLE scores (
  id SERIAL PRIMARY KEY,
  competition_id INTEGER NOT NULL,
  athlete_id INTEGER NOT NULL,
  judge_id INTEGER NOT NULL,
  action_difficulty DECIMAL(5,2),    -- 根据比赛类型: 0-30, 0-35, 或 0-50
  stage_artistry DECIMAL(5,2),       -- 0-25
  action_creativity DECIMAL(5,2),    -- 0-20, 0-15, 或 0-30
  action_fluency DECIMAL(5,2),       -- 0-15 或 0-20
  costume_styling DECIMAL(5,2),      -- 0-10
  action_interaction DECIMAL(5,2),   -- 0-15
  submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(competition_id, athlete_id, judge_id)
);
```

### 权重配置
权重配置在前端组件中硬编码：
- `components/judge/score-input-form.tsx` - `getDimensionWeight()` 函数

## 注意事项

1. **分数范围**：每个维度的分数范围等于其权重百分比（如 30% = 0-30 分）
2. **总分计算**：各维度分数直接相加，满分 100 分
3. **必填项**：所有显示的评分维度都是必填项
4. **输入验证**：系统会自动验证分数是否在有效范围内
5. **一次提交**：每位评委对每位选手只能提交一次评分，提交后无法修改
6. **实时同步**：评分提交后会实时同步到大屏幕和排行榜系统
