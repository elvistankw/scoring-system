# 评分后选手移除功能实现总结

## 功能描述
在评分界面对选手提交评分后，该选手选项从选择选手列表中移除，避免重复评分。**重要：每个评审只看到自己未评分的选手，一位评审评分后不会影响其他评审的选手列表。**

## 核心隔离机制

### 评审隔离保证
- ✅ **个人化过滤**：每个评审只看到自己未评分的选手
- ✅ **互不影响**：评审A评分后，评审B的选手列表不受影响
- ✅ **双向隔离**：无论哪个评审评分，都不会影响其他评审
- ✅ **实时更新**：评审自己评分后立即看到更新的列表

### SQL查询隔离逻辑
```sql
-- 只排除当前评审已评分的选手
SELECT * FROM athletes 
WHERE id IN (SELECT athlete_id FROM competition_athletes WHERE competition_id = ?)
AND id NOT IN (
  SELECT athlete_id FROM scores 
  WHERE judge_id = ? AND competition_id = ?  -- 关键：使用特定的judge_id
)
```

## 实现方案

### 1. 后端API修改 (`backend/controllers/athletes.controller.js`)
- 在 `getAllAthletes` 函数中添加了新的查询参数支持：
  - `judge_id`: 评审ID（必需，用于隔离）
  - `exclude_scored`: 是否排除已评分选手 (true/false)
- 当 `exclude_scored=true` 且提供了 `judge_id` 和 `competition_id` 时，API会排除该特定评审已经评分的选手

### 2. 前端Hook修改 (`hooks/use-athletes.ts`)
- 修改 `useAthletes` hook 支持新的参数：
  - `judgeId`: 评审ID
  - `excludeScored`: 是否排除已评分选手
- 自动构建正确的API查询参数，确保传入当前评审的ID

### 3. 评分页面组件修改 (`app/[locale]/(judge)/scoring/scoring-client.tsx`)
- 导入并使用 `useUser` hook 获取当前评审信息
- 在调用 `useAthletes` 时传入当前评审ID和排除已评分选手的标志
- 修改成功提示信息，明确告知选手已从列表中移除
- 更新空状态显示，当所有选手都已评分时显示相应提示

### 4. 用户体验优化
- **实时更新**: 评分提交成功后立即刷新选手列表
- **清晰反馈**: 提示信息明确告知选手已从列表移除
- **状态显示**: 选手计数显示为"X位待评分选手"
- **完成状态**: 当所有选手都已评分时显示完成提示

## 隔离性测试验证

### 测试场景1：单向隔离
- Judge 1 评分前：看到3个选手
- Judge 1 评分后：看到2个选手（减少1个）
- Judge 2 始终看到：4个选手（不受影响）
- ✅ **结果：隔离正常**

### 测试场景2：双向隔离
- 初始状态：Judge 1看到2个，Judge 2看到4个
- Judge 2 评分后：Judge 1仍看到2个，Judge 2看到3个
- ✅ **结果：双向隔离完美**

### 测试场景3：多评审并发
- 多个评审同时使用系统
- 每个评审只看到自己的未评分选手
- 互不干扰，各自独立
- ✅ **结果：并发隔离正常**
