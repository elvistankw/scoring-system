# 🚀 快速启动指南

## 问题已修复 ✅

1. **端口5000占用** - 已清理
2. **速率限制过严** - 已调整为开发模式

---

## 启动方式

### 方式1: 自动启动（推荐）

在项目根目录运行：

```bash
npm run start-all
```

这会自动启动后端（端口5000）和前端（端口3000）。

---

### 方式2: 分别启动

**终端1 - 后端：**
```bash
cd backend
npm start
```

**终端2 - 前端：**
```bash
npm start
```

---

## 如果端口被占用

运行以下命令清理端口5000：

```bash
npm run kill-port
```

---

## 访问地址

- **前端**: http://localhost:3000
- **后端API**: http://localhost:5000

---

## 速率限制（已调整）

开发模式下的限制：
- 通用API: 1000 请求/15分钟
- 管理员API: 500 请求/15分钟
- 认证API: 50 请求/15分钟
- 评分API: 30 请求/分钟
- 展示API: 200 请求/15分钟

⚠️ **生产环境需要降低这些限制！**

---

## 停止服务器

在运行的终端按 `Ctrl+C`

---

## 常用命令

```bash
# 启动所有服务
npm run start-all

# 清理端口5000
npm run kill-port

# 仅启动前端
npm start

# 仅启动后端
cd backend && npm start

# 开发模式（热重载）
npm run dev
```
