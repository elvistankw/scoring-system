# 测试选手数量显示功能

## 🧪 测试步骤

### 1. 启动服务

```bash
# 后端已在运行
# 启动前端
npm run dev
```

### 2. 访问评审控制台

1. 打开浏览器访问: `http://localhost:3000`
2. 使用评审账号登录（或创建一个评审账号）
3. 进入评审仪表板 (`/judge-dashboard`)

### 3. 查看比赛卡片

在比赛选择界面，每个比赛卡片应该显示：

```
┌─────────────────────────────────┐
│ 2025 华东赛区双人/团队赛         │
│                                 │
│ [个人赛] [👥 2]  ← 选手数量      │
│                                 │
│ 📍 华东赛区                      │
│ 📅 2025-04-15                   │
│ ✅ 可以评分                      │
└─────────────────────────────────┘
```

### 4. 验证点

- [ ] 选手数量徽章显示（紫色背景）
- [ ] 人员图标显示
- [ ] 数字正确（与实际选手数量匹配）
- [ ] 深色模式下颜色正确
- [ ] 响应式布局正常

## 🔍 如果没有显示

### 检查1: 后端API

```bash
node backend/test-judge-competitions.js
```

应该看到：
```
✅ 所有比赛都包含 athlete_count 字段
```

### 检查2: 浏览器控制台

1. 打开浏览器开发者工具 (F12)
2. 切换到 Network 标签
3. 刷新页面
4. 查找 `/api/competitions` 请求
5. 检查响应数据是否包含 `athlete_count` 字段

### 检查3: 清除缓存

如果API返回的数据来自缓存且没有 `athlete_count`:

```bash
# 方法1: 清除Redis缓存
node backend/clear-competition-cache.js

# 方法2: 重启后端
# 停止后端进程
# 重新启动
cd backend
npm start
```

## ✅ 预期结果

1. **有选手的比赛**: 显示实际数量，如 `[👥 30]`
2. **无选手的比赛**: 显示 `[👥 0]`
3. **所有比赛**: 都应该显示选手数量徽章

## 🎨 样式检查

### Light Mode
- 徽章背景: 浅紫色 (#F3E8FF)
- 文字颜色: 深紫色 (#6B21A8)

### Dark Mode
- 徽章背景: 深紫色 (#581C87)
- 文字颜色: 浅紫色 (#E9D5FF)

## 📱 响应式测试

测试不同屏幕尺寸：
- Desktop (1920x1080)
- Tablet (768x1024) - iPad
- Mobile (375x667)

徽章应该在小屏幕上正确换行。

## 🐛 常见问题

### Q: 选手数量不显示
A: 检查 `competition.athlete_count !== undefined` 条件是否满足

### Q: 数量显示为 0 但实际有选手
A: 清除缓存并重新加载数据

### Q: 样式不正确
A: 检查 Tailwind CSS 类名是否正确，确保支持 dark: 前缀

## 📞 需要帮助？

如果遇到问题，检查：
1. 后端日志 (backend/logs)
2. 浏览器控制台错误
3. Network 请求响应
4. Redis 连接状态
