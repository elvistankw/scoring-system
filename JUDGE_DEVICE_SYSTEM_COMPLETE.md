# 评委设备绑定系统 - 完整实现总结

## 🎯 系统概述

成功实现了全新的评委身份选择系统，完全改变了评委的使用方式：

**之前**: 账号 = Judge (需要登录)  
**现在**: 身份记录 = Judge (无需登录，设备绑定)

## 🔥 核心特性

### 1️⃣ 开放 Judge Landing Page
- ❌ Judge 不需要登录账号
- ✅ 任何设备打开页面都可以进入
- 👉 但不能直接评分，必须选身份

### 2️⃣ 重组 Judge 身份逻辑
- **关键变化**: 以前：账号 = Judge，现在：身份（Judge Profile） = Judge ✅
- **本质**: Judge 是一条"数据记录"，不是登录用户

### 3️⃣ 点击开始 → 选择评委身份
- **流程**: Landing Page → 点击 Start → 弹出"选择评委身份" → 选一个名字 → 进入评分

### 4️⃣ 选择身份后就可以直接使用
- 不需要密码，不需要 OTP
- 选了就能用

### 5️⃣ Admin 可以启用/停用 Judge 身份
- **数据结构**: `judges` 表包含 `id`, `name`, `status` (active/disabled)
- **行为**: disabled → 不显示/不可选，active → 可以选

### 6️⃣ 一个身份只能用在一台设备 (设备锁定)
- **核心机制**: Device Locking
- **实现**: `judge_sessions` 表记录 `judge_id` + `device_id` + `active`
- **设备识别**: localStorage 生成 UUID + 浏览器指纹

### 7️⃣ 身份占用状态显示
- **UI 行为**: 
  - Judge A ✅ 可选
  - Judge B ❌ 已被使用（灰掉）
  - Judge C ❌ 已停用

### 8️⃣ 保留 sign-in 给 admin 使用
- **双入口系统**:
  - 👤 Judge: 不登录，用 Landing Page + 选身份
  - 🛠 Admin: 正常登录（账号密码），管理 Judge 名单、启用/停用、查看评分

## 🏗️ 技术实现

### 数据库架构

#### 新的 judge_sessions 表
```sql
CREATE TABLE judge_sessions (
    id SERIAL PRIMARY KEY,
    judge_id INTEGER NOT NULL REFERENCES judges(id),
    device_id VARCHAR(255) NOT NULL, -- UUID generated by frontend
    judge_name VARCHAR(255) NOT NULL,
    judge_code VARCHAR(50) NOT NULL,
    ip_address INET,
    user_agent TEXT,
    browser_fingerprint TEXT,
    started_at TIMESTAMP DEFAULT NOW(),
    last_activity TIMESTAMP DEFAULT NOW(),
    expires_at TIMESTAMP DEFAULT (NOW() + INTERVAL '8 hours'),
    ended_at TIMESTAMP NULL,
    is_active BOOLEAN DEFAULT true
);

-- 设备锁定：确保每个评委只能有一个活跃会话
CREATE UNIQUE INDEX idx_unique_active_judge 
ON judge_sessions (judge_id) 
WHERE is_active = true;
```

#### 数据库函数
- `cleanup_expired_judge_sessions()`: 自动清理过期会话
- `is_judge_available(judge_id)`: 检查评委是否可用
- `get_available_judges()`: 获取所有评委及其可用状态

### 后端 API 端点

#### 公共端点 (无需认证)
- `GET /api/judges/available` - 获取可用评委列表
- `POST /api/judges/select-identity` - 选择评委身份
- `GET /api/judges/current-session?deviceId=xxx` - 获取当前会话
- `POST /api/judges/end-session` - 结束会话
- `POST /api/judges/extend-session` - 延长会话

#### 管理端点 (需要 Admin 认证)
- `GET /api/judges` - 获取所有评委 (含使用状态)
- `POST /api/judges` - 创建评委
- `PUT /api/judges/:id` - 更新评委
- `DELETE /api/judges/:id` - 删除评委
- `POST /api/judges/:id/toggle-active` - 启用/停用评委
- `GET /api/judges/stats` - 获取统计信息

### 前端组件

#### 设备管理
- `lib/device-manager.ts`: 设备ID生成和管理
- 自动生成 UUID 作为设备标识
- 生成浏览器指纹作为额外验证
- localStorage 持久化存储

#### 会话管理
- `hooks/use-judge-session.ts`: 评委会话管理 Hook
- 自动获取可用评委
- 处理身份选择
- 管理会话状态
- 自动刷新和延长会话

#### UI 组件
- `components/judge/judge-landing-client.tsx`: 新的 Landing Page
- `components/judge/judge-identity-modal.tsx`: 身份选择弹窗
- 支持深色模式
- 响应式设计 (平板优化)
- 双语支持
